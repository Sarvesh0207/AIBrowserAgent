# Docs

- **HEADLESS_EVALUATION.md** — Headless evaluation document (generated by the steps below)

---

## How to generate the Headless evaluation document (step by step)

### Step 1: Confirm URLs to test

Edit `data/sites_to_test.csv`: one URL per line, with header row `url`. Example:

```csv
url
https://example.com
https://www.wikipedia.org/
https://www.uic.edu/
```

### Step 2: Run benchmark (generate report CSV)

From the project root:

```bash
python main.py benchmark --csv data/sites_to_test.csv
```

- The script visits each URL in headless mode and records success or failure.
- Output: `outputs/reports/headless_report_<timestamp>.csv`.

### Step 3: Generate the evaluation document (Markdown)

Then run:

```bash
python main.py benchmark-doc
```

- Uses the **latest** `headless_report_*.csv` and generates `docs/HEADLESS_EVALUATION.md`.
- To use a specific report:  
  `python main.py benchmark-doc --report outputs/reports/headless_report_20250222T120000Z.csv`
- To set output path:  
  `python main.py benchmark-doc --out docs/my_eval.md`

### Step 4: Add categorization and findings (manual)

Open `docs/HEADLESS_EVALUATION.md` and fill in the **"Categorization / Findings (edit manually)"** section, for example:

- Which site types tend to pass or fail (e.g. e‑commerce, news).
- Common error types (e.g. `ERR_HTTP2_PROTOCOL_ERROR`, timeout).

After these steps you have a complete headless evaluation document to share.
